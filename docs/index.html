Everything you need for protein design

From parallel MCMC sampling to 3D structure visualization — Proteus provides a complete computational protein design pipeline.

⚡

Parallel MCMC Engine

5-chain parallel tempering with Metropolis-Hastings acceptance. Configurable temperatures from 0.5 to 10.0 for optimal exploration-exploitation balance.

🧬

ESM-2 Guided Proposals

Biologically plausible mutations guided by ESM-2 protein language model embeddings. Point substitutions, block replacements, and LLM jumps.

🎯

Energy-Based Scoring

Multi-objective energy function: binding affinity, stability, solubility, BBB penetration. BLOSUM62-informed and customizable weights.

📊

Real-Time Progress

WebSocket streaming of chain energies, acceptance rates, and best candidates. Live Recharts visualizations update as MCMC runs.

🔬

3D Structure Viewer

Integrated PDBe Molstar viewer for interactive protein structure exploration. Designed mutations highlighted on the target protein.

📋

Regulatory Compliance

Complete audit trail, RBAC (fellow/mentor/admin), data retention policies, dual-use research screening, and HIPAA-aligned logging.

Get Started in Minutes

Local development with Docker Compose or manual setup.

🐳 Docker (Recommended)

# Start the full stack
docker-compose up --build

# Frontend: http://localhost:5173
# API: http://localhost:8000
# Docs: http://localhost:8000/docs

⚡ Manual Setup

# Backend
cd backend && python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload

# Frontend
cd frontend && npm install
npm run dev

📊 Run Your First Design

# Register a user
curl -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"securepass","full_name":"Researcher"}'

# Start a design run against EGFRvIII
curl -X POST http://localhost:8000/api/v1/runs \
  -H "Authorization: Bearer <token>" \
  -d '{"target_name":"EGFRvIII","num_chains":5,"steps_per_chain":1000}'

# Watch progress via WebSocket
wscat -c ws://localhost:8000/api/v1/runs/<run_id>/ws
